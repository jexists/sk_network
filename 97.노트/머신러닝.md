## 핵심 개념

| 개념 | 설명 |
|------|------|
| **선형회귀** | 직선(y = ax + b)으로 데이터 설명 |
| **비선형회귀** | 곡선으로 데이터 설명 |
| **잔차** | 실제값 - 예측값 (어긋남) |
| **최소제곱법(LSE)** | 잔차² 합이 최소인 직선 찾기 → 학습 원리 |

## 주요 회귀 알고리즘

### 선형 회귀 (Linear Regression)
- **특징**: y = ax + b 직선으로 데이터 적합. 가장 기본적인 회귀 모델
- **학습 원리**: 최소제곱법(LSE) — 잔차² 합 최소화
- **사용**: 변수 간 선형 관계가 명확할 때, 해석이 중요한 경우
- **단점**: 이상치에 민감, 비선형 관계 표현 불가

### 다항 회귀 (Polynomial Regression)
- **특징**: x², x³ 등 고차항 추가로 곡선 표현. 선형 회귀의 확장
- **학습 원리**: LSE (입력 변수를 변환 후 선형 회귀 적용)
- **사용**: 데이터가 곡선 패턴일 때
- **단점**: 차수가 높아질수록 과적합 위험

### 비선형 회귀 (NonLinear Regression)
- **특징**: 지수함수, 로그함수 등 비선형 수식으로 데이터 적합
- **학습 원리**: 비선형 최적화 (경사하강법 등)
- **사용**: 성장 곡선, 포화 곡선처럼 명확한 비선형 수식이 있을 때
- **단점**: 초기값 설정에 민감, 수렴 어려울 수 있음

### 릿지 회귀 (Ridge)
- **특징**: 선형 회귀 + L2 정규화 (계수² 합에 패널티)
- **학습 원리**: LSE + λΣβ² 최소화
- **사용**: 변수가 많고 다중공선성이 있을 때, 모든 변수를 유지하고 싶을 때
- **단점**: 변수를 0으로 만들지 않아 변수 선택 불가

### 라소 회귀 (Lasso)
- **특징**: 선형 회귀 + L1 정규화 (계수 절댓값 합에 패널티)
- **학습 원리**: LSE + λΣ|β| 최소화
- **사용**: 중요하지 않은 변수를 0으로 만들어 자동 변수 선택
- **단점**: 상관된 변수 중 하나만 선택하는 경향

### ElasticNet
- **특징**: L1 + L2 정규화 혼합 (Ridge + Lasso 결합)
- **학습 원리**: LSE + λ₁Σ|β| + λ₂Σβ² 최소화
- **사용**: 변수가 매우 많고 그룹 구조가 있을 때, Lasso와 Ridge 사이에서 균형 필요할 때
- **단점**: 하이퍼파라미터(λ₁, λ₂) 두 개 튜닝 필요

### 로지스틱 회귀 (Logistic Regression)
- **특징**: 이름은 회귀지만 분류 모델. 확률값(0~1) 출력
- **학습 원리**: MLE (최대우도추정), 시그모이드 함수 사용
- **사용**: 이진 분류 (스팸/정상, 양성/음성), 확률 해석이 필요할 때
- **단점**: 선형 결정 경계만 표현 가능

### SGD 회귀 (Stochastic Gradient Descent)
- **특징**: 전체 데이터 대신 샘플 하나씩 뽑아 경사하강법 적용
- **학습 원리**: 확률적 경사하강법 — 배치 SGD보다 빠름
- **사용**: 데이터가 매우 많아 전체를 메모리에 올릴 수 없을 때, 온라인 학습
- **단점**: 수렴이 불안정, 학습률 설정 중요

### 결정 트리 (Decision Tree Regression)
- **특징**: if/else 규칙으로 데이터 분할, 각 리프에서 평균값 예측
- **학습 원리**: 불순도 감소 (MSE 최소화) 기준으로 분기
- **사용**: 비선형 관계, 규칙을 사람이 이해해야 할 때
- **단점**: 과적합 쉬움, 데이터 변화에 민감

### 랜덤 포레스트 (Random Forest)
- **특징**: 여러 결정 트리를 만들어 평균값으로 예측 (배깅 앙상블)
- **학습 원리**: 부트스트랩 샘플링 + 랜덤 변수 선택
- **사용**: 일반적인 회귀 문제, 과적합 방지가 필요할 때
- **단점**: 학습/예측 속도 느림, 해석 어려움

### XGBoost
- **특징**: 약한 트리를 순차적으로 쌓아 오류를 보완 (부스팅 앙상블)
- **학습 원리**: 경사 부스팅 — 잔차를 다음 트리가 학습
- **사용**: 정형 데이터 예측 대회, 높은 성능이 필요할 때
- **단점**: 하이퍼파라미터 많아 튜닝 복잡, 과적합 주의

### SVR (Support Vector Regression)
- **특징**: 마진(ε) 안에 최대한 많은 점을 포함하는 함수 학습
- **학습 원리**: 커널 트릭으로 고차원 변환 후 선형 회귀
- **사용**: 이상치가 있거나 데이터가 적을 때, 비선형 관계
- **단점**: 대용량 데이터에 느림, 파라미터(C, ε, 커널) 튜닝 필요

### 로버스트 회귀 (Robust Regression)
- **특징**: 이상치 영향을 줄인 선형 회귀 (Huber Loss 등 사용)
- **학습 원리**: 이상치에 작은 가중치 부여
- **사용**: 데이터에 이상치가 많고 제거하기 어려울 때
- **단점**: 일반 선형 회귀보다 계산 복잡

## 과적합 / 과소적합

```
과소적합 (Underfitting)     적절     과적합 (Overfitting)
모델이 너무 단순            ✅        모델이 너무 복잡
훈련/테스트 모두 성능 낮음           훈련 성능↑, 테스트 성능↓
```

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| **과소적합** | 모델 너무 단순, 변수 부족 | 복잡한 모델 사용, 변수 추가 |
| **과적합** | 모델 너무 복잡, 데이터 부족 | 정규화(L1/L2), 데이터 늘리기, 교차검증 |

**Bias-Variance Tradeoff**: 모델 복잡도가 올라가면 bias↓, variance↑ — 둘의 균형점 찾기

## 전처리 (Preprocessing)

### 스케일링 (수치형 변수)

| 방법 | 수식 | 사용 |
|------|------|------|
| **StandardScaler** | (x - 평균) / 표준편차 | 정규분포 가정, 대부분의 경우 |
| **MinMaxScaler** | (x - min) / (max - min) → 0~1 | 범위가 중요할 때 |
| **RobustScaler** | (x - 중앙값) / IQR | 이상치가 많을 때 |

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)  # 훈련셋으로 fit
X_test = scaler.transform(X_test)        # 테스트셋은 transform만
```

### 인코딩 (범주형 변수)

| 방법 | 설명 | 사용 |
|------|------|------|
| **Label Encoding** | 카테고리 → 정수 (0, 1, 2...) | 트리 모델 |
| **One-Hot Encoding** | 카테고리 → 0/1 열로 분리 | 선형 모델 |

```python
from sklearn.preprocessing import LabelEncoder, OneHotEncoder
import pandas as pd

# Label Encoding
le = LabelEncoder()
df['col'] = le.fit_transform(df['col'])

# One-Hot Encoding
df = pd.get_dummies(df, columns=['col'])
```

### 결측치 처리

```python
df.isnull().sum()           # 결측치 확인
df.fillna(df.mean())        # 평균으로 대체
df.dropna()                 # 행 제거

from sklearn.impute import SimpleImputer
imputer = SimpleImputer(strategy='mean')  # 'median', 'most_frequent'
```

from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression

# 1. 데이터 분할
X_train, X_test, y_train, y_test = train_test_split(

)

# 2. 모델 학습
model = LinearRegression()

## 교차검증 (Cross Validation)

단순 train/test 분할은 운에 따라 성능이 달라질 수 있음 → 교차검증으로 신뢰도 높이기

```
K-Fold (k=5 예시):
[1][2][3][4][5]
 T  T  T  T  V  → 1회
 T  T  T  V  T  → 2회
 T  T  V  T  T  → 3회
 T  V  T  T  T  → 4회
 V  T  T  T  T  → 5회
평균 성능 사용
```

```python
from sklearn.model_selection import cross_val_score, KFold

kf = KFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=kf, scoring='r2')
print(scores.mean(), scores.std())
```

## 분류 알고리즘

| 알고리즘 | 특징 | 사용 |
|----------|------|------|
| **로지스틱 회귀** | 선형 결정 경계, 확률 출력 | 이진/다중 분류, 해석 필요할 때 |
| **KNN** | 가까운 k개 이웃 다수결 | 데이터 적고 단순할 때 |
| **SVM** | 마진 최대화 결정 경계 | 고차원, 데이터 적을 때 |
| **결정 트리** | if/else 규칙 분류 | 해석 필요할 때 |
| **랜덤 포레스트** | 트리 앙상블 (배깅) | 일반적인 분류 문제 |
| **XGBoost** | 트리 앙상블 (부스팅) | 고성능 필요할 때 |
| **Naive Bayes** | 베이즈 확률 기반 | 텍스트 분류 |

### 분류 성능 지표

| 지표 | 수식 | 의미 |
|------|------|------|
| **Accuracy** | (TP+TN) / 전체 | 전체 정확도 (불균형 데이터엔 부적합) |
| **Precision** | TP / (TP+FP) | 양성 예측 중 실제 양성 비율 |
| **Recall** | TP / (TP+FN) | 실제 양성 중 양성으로 예측한 비율 |
| **F1 Score** | 2 × (P×R) / (P+R) | Precision과 Recall의 조화평균 |
| **ROC-AUC** | ROC 곡선 아래 면적 | 0.5~1, 높을수록 좋음 |

```
헷갈릴 때:
- 스팸 필터 → Precision 중요 (정상 메일을 스팸으로 잘못 분류하면 안 됨)
- 암 진단 → Recall 중요 (암 환자를 놓치면 안 됨)
```

```python
from sklearn.metrics import classification_report, confusion_matrix

print(classification_report(y_test, y_pred))
print(confusion_matrix(y_test, y_pred))
```

## 클러스터링 (비지도 학습)

정답(label) 없이 데이터를 유사한 그룹으로 묶기

| 알고리즘 | 특징 | 사용 |
|----------|------|------|
| **K-Means** | k개 중심점 기준으로 군집화 | 군집 수를 알 때, 구형 군집 |
| **DBSCAN** | 밀도 기반 군집화 | 군집 수 모를 때, 이상치 탐지 |

```python
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=3, random_state=42)
kmeans.fit(X)
labels = kmeans.labels_  # 각 샘플의 군집 번호
```

**엘보우 기법**: K-Means에서 최적 k 찾기 — k를 늘리면서 inertia(군집 내 분산) 감소폭이 꺾이는 지점 선택
